# Exec Plan Template

## Objective

State the user-visible or maintainer-visible outcome.

## Context

Summarize the relevant repository area, constraints, and prior behavior.

## Constraints

List non-negotiable rules, environment limitations, compatibility targets, or release requirements.

## Milestones

1. Discovery
   Files, scripts, and behaviors to inspect first.
2. Implementation
   Concrete changes to make.
3. Validation
   Commands, builds, or E2E runs required before closure.
4. Distribution Or Release Follow-Up
   Rebuilds, deploys, release-repo sync, or docs updates if needed.

## Validation

List the exact commands and evidence required to treat the work as complete.

## Risks And Open Questions

List unresolved assumptions, likely failure modes, or decisions that may require confirmation.
